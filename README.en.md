## Quick Start

**Paste this into your Claude Code and it will handle everything:**

```
帮我切换 Claude Code 的 /buddy 宠物。

步骤：
1. 先 clone 这个仓库（如果还没有的话）：git clone https://github.com/Zhao73/claude-buddy-reroll.git /tmp/buddy-reroll
2. 运行 node /tmp/buddy-reroll/switch.js 查看所有 18 种传说闪光宠物的外观和属性
3. 把所有宠物的名字、外观、属性展示给我看，问我想要哪一个
4. 我选好后，运行 node /tmp/buddy-reroll/switch.js <我选的物种名> 来切换
5. 告诉我需要怎么重启 Claude Code（需要用 CLAUDE_CODE_OAUTH_TOKEN 环境变量启动）

注意：如果我还没有 OAuth token，先在单独的终端运行 claude setup-token 获取。
```

> **That's it!** Claude Code will clone the repo, show you all 18 pets, let you pick one, and switch it for you.

Or if you prefer doing it yourself, use the interactive switcher:

```bash
git clone https://github.com/Zhao73/claude-buddy-reroll.git
cd claude-buddy-reroll

# Browse all 18 legendary shiny pets
node switch.js

# Switch to your favorite
node switch.js dragon

# Check current status
node switch.js --status

# Restore original buddy
node switch.js --restore
```

## How It Works

Claude Code's `/buddy` system uses a **Bones + Soul** architecture:

| Layer | Determines | Seed | Mutable? |
|-------|-----------|------|----------|
| **Bones** (appearance) | Species, Rarity, Eyes, Hat, Shiny, Stats | `accountUuid` or `userID` | See below |
| **Soul** (personality) | Name, Personality description | Generated by Claude at hatch | Yes (delete `companion` in config) |

The **Bones** layer hashes your identity with [FNV-1a](https://en.wikipedia.org/wiki/Fowler%E2%80%93Noll%E2%80%93Vo_hash_function) + [Mulberry32 PRNG](https://gist.github.com/tommyettinger/46a874533244883189143505d203312c), using the seed:

```
seed = FNV1a(userId + "friend-2026-401")
```

### The Loophole

When you log in with the `CLAUDE_CODE_OAUTH_TOKEN` environment variable, Claude Code **does not write `accountUuid`** into `~/.claude.json`. Without `accountUuid`, `/buddy` falls back to the `userID` field — which you can freely edit.

## 5 Stats

Each buddy has five core attributes. One is randomly boosted (**peak**), one is nerfed (**valley**), the rest are **normal**.

| Stat | Description |
|------|-------------|
| **DEBUGGING** | Bug-finding instinct |
| **PATIENCE** | Tolerance for your mistakes |
| **CHAOS** | Tendency to cause mayhem |
| **WISDOM** | Depth of insight |
| **SNARK** | Sass level in comments |

### Stat Generation Formula

| Type | Formula | Legendary Range |
|------|---------|-----------------|
| **Peak** | `min(100, base + 50 + random(0~29))` | 100 |
| **Valley** | `max(1, base - 10 + random(0~14))` | 40 ~ 54 |
| **Normal** | `base + random(0~39)` | 50 ~ 89 |

Peak and valley stats are randomly selected from the 5 attributes using the same PRNG sequence.

## Step-by-Step Guide

### Prerequisites

- Claude Code installed (`npm install -g @anthropic-ai/claude-code`)
- Claude Max / Pro subscription (OAuth login)
- Node.js 18+

### Step 1: Get Your OAuth Token

Open a **separate terminal** (not inside Claude Code):

```bash
claude setup-token
```

Copy the token that starts with `sk-ant-oat01-...`.

### Step 2: Backup Your Config

```bash
cp ~/.claude.json ~/.claude.json.backup
```

### Step 3: Remove `accountUuid` and Set Target `userID`

Pick a `userID` from the [Pre-rolled Collection](#pre-rolled-legendary-shiny-collection) above, then run:

```bash
node apply.js PASTE_YOUR_CHOSEN_ID_HERE
```

Or manually:

```bash
node -e "
const fs = require('fs');
const path = process.env.HOME + '/.claude.json';
const config = JSON.parse(fs.readFileSync(path, 'utf8'));

// Remove accountUuid so /buddy falls back to userID
if (config.oauthAccount) delete config.oauthAccount.accountUuid;

// Set your chosen userID (example: legendary shiny dragon)
config.userID = 'PASTE_YOUR_CHOSEN_ID_HERE';

// Clear companion to trigger re-hatch
delete config.companion;

fs.writeFileSync(path, JSON.stringify(config, null, 2));
console.log('Done!');
"
```

### Step 4: Launch with OAuth Token

```bash
export CLAUDE_CODE_OAUTH_TOKEN="sk-ant-oat01-YOUR_TOKEN_HERE"
claude
```

### Step 5: Hatch!

Inside Claude Code, type:

```
/buddy
```

Your new Legendary Shiny buddy will hatch with a fresh name and personality.

### Rollback

If anything goes wrong:

```bash
node apply.js --restore
# or manually:
cp ~/.claude.json.backup ~/.claude.json
```

## Technical Details

### Algorithm

```
input   = userID + "friend-2026-401"
hash    = FNV-1a(input)             → 32-bit integer
rng     = Mulberry32(hash)          → deterministic PRNG

rarity  = weightedPick(rng)         → [60% common ... 1% legendary]
species = uniformPick(rng, 18)      → one of 18 species
eye     = uniformPick(rng, 6)       → one of 6 eye styles
hat     = uniformPick(rng, 8)       → one of 8 hats (common = none)
shiny   = rng() < 0.01             → independent 1% chance

peak    = pick(rng, STATS)          → one stat boosted
valley  = pick(rng, STATS) ≠ peak  → one stat nerfed
stats   = for each stat:
            peak   → min(100, base + 50 + floor(rng() * 30))
            valley → max(1,   base - 10 + floor(rng() * 15))
            normal → base + floor(rng() * 40)
```

### Why `accountUuid` Bypass Works

```
Normal login:       accountUuid → written to ~/.claude.json → /buddy uses it
OAuth token login:  accountUuid → NOT written               → /buddy uses userID
```

Since `userID` is just a local field in `~/.claude.json`, you can set it to any string. Different strings produce different pets through the hash function.
